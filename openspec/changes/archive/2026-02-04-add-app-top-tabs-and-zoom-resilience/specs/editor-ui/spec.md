## ADDED Requirements

### Requirement: 应用顶级标签页

应用 SHALL 在根布局左侧提供竖向排列的顶级标签栏，包含「提示词管理」「MCP 中心」「提示词市场」三个标签。用户点击某一标签时，主内容区 SHALL 切换为对应页面，且现有功能（模板列表与编辑、MCP 管理、市场浏览与安装）均可用。

#### Scenario: 点击顶级标签切换主内容

- **GIVEN** 应用已加载
- **WHEN** 用户点击左侧标签栏中的「提示词管理」
- **THEN** 主内容区显示当前的提示词管理视图（模板侧栏 + 发现/编辑区域）
- **AND THEN** 模板列表、编辑、发现等现有交互均正常

#### Scenario: 切换到 MCP 中心

- **GIVEN** 应用已加载
- **WHEN** 用户点击左侧标签栏中的「MCP 中心」
- **THEN** 主内容区显示 MCP 资源中心完整内容（与当前 McpManager 能力一致）
- **AND THEN** 用户可管理 MCP 服务、技能等而无需通过弹窗进入

#### Scenario: 切换到提示词市场

- **GIVEN** 应用已加载
- **WHEN** 用户点击左侧标签栏中的「提示词市场」
- **THEN** 主内容区显示提示词市场完整内容（与当前 MarketView 能力一致）
- **AND THEN** 用户可搜索、安装市场提示词而无需通过弹窗进入

### Requirement: 缩放时关键 UI 抗乱序

在浏览器或系统对页面进行缩放时，关键 UI 区域（顶级标签栏、模板侧栏标题与主要按钮、编辑器工具栏、主要列表项文字等）SHALL 保持可读与可操作，不出现严重文字换行错乱、重叠或不可点击。系统 SHALL 通过为上述关键元素指定最小字号或最小尺寸（如 CSS min-width/min-height 或 clamp）来实现抗乱序。

#### Scenario: 缩放后标签栏仍可用

- **GIVEN** 用户将页面缩放至 80% 或 110%
- **WHEN** 查看左侧顶级标签栏
- **THEN** 标签仍能清晰辨认且可点击切换
- **AND THEN** 不出现标签文字严重重叠或截断至不可读

#### Scenario: 缩放后侧栏与工具栏可操作

- **GIVEN** 用户将页面缩放至 90% 或 120%
- **WHEN** 使用提示词管理视图中的模板侧栏与编辑器工具栏
- **THEN** 主要按钮与列表项文字保持可读、可点击
- **AND THEN** 不出现因缩放导致的严重换行错乱或元素重叠
