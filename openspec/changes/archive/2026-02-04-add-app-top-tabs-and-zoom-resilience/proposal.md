# Change: 应用顶级标签页与缩放抗乱序

## Why

1. **导航统一**：当前「提示词管理」主流程、MCP 资源中心、提示词市场分别通过侧边栏按钮或全屏弹窗进入，入口分散且层级不清晰。将三者作为应用顶级标签页并在左侧竖向排列，可统一入口、降低认知负担。
2. **缩放体验**：用户通过浏览器或系统缩放页面时，部分关键 UI（如侧边栏标题、按钮、编辑器区域）可能出现文字换行错乱、元素重叠或过小难以点击。需要对关键元素约定最小可缩放尺寸或最小字号，保证缩放后排版仍可用。

## What Changes

- 在应用根布局左侧新增**竖向顶级标签栏**，包含三个标签：提示词管理、MCP 中心、提示词市场。点击切换后主内容区展示对应页面，现有功能保持不变（提示词管理 = 当前侧栏 + 发现/编辑；MCP 中心 = 当前 McpManager 内容；提示词市场 = 当前 MarketView 内容）。
- MCP 中心与提示词市场由「全屏弹窗」改为**顶级标签下的主内容区**展示，仅保留在内容区内的关闭/返回入口（若设计需要）。
- 引入**缩放抗乱序**策略：对关键 UI 元素（如顶级标签栏、侧边栏标题与按钮、编辑器工具栏、主要列表项文字等）应用最小字号或最小尺寸（如 `min-width`/`min-height` 或 `min-font-size`），或使用 `clamp()`/`min()` 等 CSS 方式，使在常见缩放比例下不出现严重换行错乱或不可用。

## Impact

- **Affected specs**: editor-ui（应用壳与布局、缩放策略均属编辑器/应用 UI 范畴）。
- **Affected code**: `prompt_fill/src/App.jsx`（根布局、顶级 tab 状态与内容切换），`TemplatesSidebar.jsx`（可能移除或迁移 MCP/市场入口至顶级栏），`McpManager.jsx`、`MarketView.jsx`（由弹窗改为嵌入主内容区），可选全局样式或 `index.css` / 组件内样式（最小字号/尺寸常量）。
